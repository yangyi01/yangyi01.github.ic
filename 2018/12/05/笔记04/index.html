December 5, 2018 本文总阅读量次 
 1.列约束
  （1）主键约束——PRIMARY KEY
    (2) 唯一约束 UNIQUE
  （3）非空约束     NOT NULL
          生命力非空约束的列上不能插入
  （4）默认值约束——DEFAUITk
      可以使用DEFAULT 关键字声明默认值，有两种方式可以应用默认值
    例如：
     INSERT INTO xz_laptop VALUES (40,'苹果'，DEFAULT);
     INSERT INTO xz_laptop VALUES (40,'苹果'，);
  （5） 检查约束——CHAECK
     检查约束可以对插入的数据进行检验
     CREATE TABLE student (
             age TINYINT  CHECK(age>=18 AND age<=60)
);
     MySQL 不支持检查约束，会降低数据的插入速度。
   （6）外键约束 —— FOREIGN KEY
声明了外键约束的列，取值必须在另一个表的主键列上出现过，
列类型要保持一致，取值可以是NULL
        FOREIGN  KEY(familyId）REFERENCES xz_laptop_family(fid)
2.MySQL 中的自增列
       AUTO_INCREMENT：自动增长   假入了一个列声明了自增列，无需手动赋值，之间设置为NULL
    会获取当前的最大值，然后加1插入。
                 注意：
                          自增列允许手动赋值
                          只适用于整数型的主键列上
 3. 简单查询
 （1）查询特定的列
   示例：查询所有员工的姓名‘工资、生日
  SELECT ename,salary,brithday FROM emp;
 （2）查询所有的列
          SELECT * FROM emp；
      SELECT eid,ename,sex,birthday,salary,deptId FROM emp;    
  （3）给列起别名
    示例：查询所有员工的姓名和工资，使用汉字的别名显示
    SELECT ename AS 姓名,salary AS 工资 FROM emp; 
         SELECT  eid AS 编号,ename AS 姓名,sex AS 性别,birthday AS 生日 FROM emp;
   注意：AS 关键字是可以省略的，但是需要保留空格。
  （4）只显示不同的记录/合并相同的记录
       示例： 查询出员工都在哪些部门
      SELECT DISTINCT deptId FROM emp;
      SELECT DISTINCT sex FROM emp;
  （5）在查询时执行计算
      示例：计算2*3/4-8+10
   SELECT ename,salary*12 FROM emp;
  SELECT ename 姓名,(salary+500)*12+5000 AS 年薪 FROM emp;
   （6）对查询的结果集进行排序
         示例：查询所有的部门，对结果按照部门编号升序排序
          SELECT * FROM dept ORDER BY did ASC;                                   #ascendant   向上的；
        示例：查询所有的部门，对结果按照部门编号降序排序             
          SELECT * FROM dept ORDER BY did DESC;                                 #descendant   向下的；
          SELECT * FROM emp ORDER BY birthday DESC;  
          SELECT * FROM emp ORDER BY salary DESC,ename;
          ORDER BY   可以按照 数值  日期 时间 字符串来排序，
          默认是按照ASC升序排列。
     （7）条件查询                               
          示例：查询出编号为5的员工所有信息
          SELECT *FROM emp WHERE eid=5;
          SELECT eid,salary,birthday FROM emp WHERE ename='king';
          SELECT * FROM  emp WHERE deptId=20;
          SELECT * FROM  emp WHERE salary>=5000;
          >=   <=   <  > =  !=(不等于）
          SELECT * FROM  emp WHERE deptId!=10;
          SELECT * FROM  emp WHERE deptId IS null;                                        null只能用  IS 
          SELECT * FROM  emp WHERE deptId    IS  NOT null;
          SELECT * FROM  emp WHERE salary>6000 AND sex=0;                        用 AND(并且） 连接

          SELECT * FROM  emp WHERE salary>=5000 AND salary<=7000; 
          SELECT * FROM  emp WHERE salary BETWEEN 5000 AND 7000;       (NOT) BETWEEN ..... AND (不）在...之间 
          SELECT * FROM  emp WHERE salary<=5000 OR salary>=7000;                OR（或者） 连接 满足两个条件
          SELECT * FROM  emp WHERE salary NOT BETWEEN 5000 AND 7000;                                                                    SELECT * FROM emp WHERE deptId IN (10,30);     NOT  IN    不 在...         IN 在....


            IS NULL / IS NOT NULL
            AND /OR 
           BETWEEN.. AND.../ NOT BETWEE...AND...
           IN （）/NOT IN （） 
     注意：删除、更改、查询都可以结合条件查询。
      （8）模糊条件查询
             示例：查询出姓名中含有字母e的员工信息
           SELECT * FROM emp WHERE ename LIKE '%e%';    含有e的                      %  表示可以有任意一个字符
           SELECT * FROM emp WHERE ename LIKE '%e';      位数为e的
           SELECT * FROM emp WHERE ename LIKE '%e_';    倒数第二个为e的
             SQL中提供了两个模糊查询的字符
             % 可以匹配任意多个字符  >=0
             _   可以匹配任意一个字符  =1
           注意：以上 两个匹配不能“=”使用，必须和“LIKE”使用。
    （9） 分页查询
          假如查询的结果集中有太多的数据，一次显示不完，可以分页显示。
              需要有两个条件   ：当前的页码、每页的数据量
         SELECT * FROM emp  LIMIT start,count;      
           start :是一个数字，表示从结果集中的哪一条开始读取，第一条称为0.                   
        每一页开始的算法           
          start=（页码-1）*count
         count：是一个数字，最多读取的行数
假设每一页显示五条记录
 第一页：SELECT* FROM emp LIMIT 0,'5';
